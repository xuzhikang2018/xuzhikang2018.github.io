1. 结构

ODIN是一种在线学习数字脉冲神经形态处理器，在Universitécatholiquede Louvain（UCLouvain）的28-nm FDSOI CMOS中设计和制作原型，于2019年在 IEEE Transactions on Biomedical Circuits and Systems journal上发表。ODIN基于单个256神经元64k突触交叉神经突触核心，其交叉架构和框图分别在1.1节和1.2节中描述。ODIN具有以下主要功能：

  • 突触嵌入基于脉冲依赖突触可塑性（SDSP）的在线学习
  • 神经元可以单独选择作为标准的LIF模型或作为现象学Izhikevich（IZH）神经元模型来再现20 Izhikevich行为。

因此，ODIN是用于边缘学习的多功能实验平台，同时展示（i）记录神经元和突触密度，与之前提出的所有脉冲神经网络（SNN）相比较，以及（ii）先前提出的数字中每个突触操作的最低能量SNNS。

1.1 Cross architecture

图1 - ODIN时间复用的交叉开关架构和由输入AER链路处理的关键事件类型。

ODIN模拟图1所示的交叉结构，其中每个神经元通过在线学习突触连接到交叉阵列中的所有256个神经元。因此，交叉结构中处理器中包括256×256 = 64k的突触。为了减少电路资源，通过时间复用来共享神经元和突触的更新逻辑：所有神经元和突触状态都存储在SRAM存储器中。来自/到交叉开关架构的输入和输出事件通过地址事件表示（AER）总线处理：

图2（a）带有主要verilog文件的ODIN框图（b）完整的verilog文件树，说明不同模块之间的依赖关系

ODIN SNN处理器框图的概述如图2（a）所示，对应块的verilog文件在图2（a）中以橙色突出显示。src文件夹中可用的详细verilog文件树如图2（b）所示。关于ODIN的体系结构和时间复用操作的更多细节可以在相应论文中找到。
neuron.v 时间复用的256个神经元,各个状态和参数存储在4kB单端口中SRAM中，并在HDL中对其进行行为描述
synaptic_core.v 时分复用的64k个突触，各个状态和参数存储在32kB单端口中SRAM中，并在HDL中对其进行行为描述
sheduler.v 神经元和突触更新逻辑的时间复用由控制器处理
controller.v 来自输入AER总线的内部和外部事件由调度器处理。
spi.v SPI总线允许配置控制寄存器并访问单个神经元和突触状态。

2.interfaces and commands

除了复位和时钟输入，顶层文件ODIN.v包含三个主要接口：SPI总线（第2.1节）和输入和输出AER总线（第2.2节）。

图3 - （a）写和（b）读操作的20位SPI时序图。

ODIN通过以下接口实现20位从SPI bus：

Pin 端口描述 位宽 描述
SCK 输入 1-bit 由 SPI master产生的时钟.
MOSI 输入 1-bit Master 输出, slave 输入.
MISO 输出 1-bit Master 输入, slave 输出.

使用SPI总线时，应声明SPI_GATE_ACTIVITY配置寄存器（第4节）。为确保正常工作，SCK SPI时钟的工作频率应至少比ODIN的时钟频率小4倍。SPI写和读操作遵循图4和图5中所示的时序图。分别如图3（a）和3（b）所示：在主机发送与该地址相关的数据（写入）或从ODIN（读取）接收的数据之前，首先由SPI主机发送20位地址字段。根据20位地址字段a的内容，SPI总线可用于访问配置寄存器，神经元存储器或突触存储器，如下所示：

R (a<19>) W (a<18>) cmd<1:0> (a<17:16>) addr<15:0> (a<15:0>) Description
N/A N/A 00 {conf_addr<15:0>} 配置寄存器写入地址 conf_addr.
1 0 01 {n/a,byte_addr<3:0>,word_addr<7:0>} 读入神经元memory（256个128位字）。搜索来自word_addr <7：0>的字节byte_addr<3：0>。
0 1 01 {n/a,byte_addr<3:0>,word_addr<7:0>} 写入神经元存储器（256个128位字）。写入字 word_addr<7：0> 的字节 byte_addr<3：0>。
1 0 10 {n/a,byte_addr<1:0>,word_addr<12:0>} 读取突触记忆（8192 32位字）。检索来自 word_addr <12：0>的字节byte_addr<1：0>。
0 1 10 {n/a,byte_addr<1:0>,word_addr<12:0>} 写入突触记忆（8192 32位字）。字节 byte_addr <1：0> 来自字 word_addr <12：0> 被写入。
